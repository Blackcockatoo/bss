# MetaPet Schools Accessibility and Inclusion Review

## Current inclusion posture

The pilot version is designed for short supervised sessions with plain language, low setup overhead, and multiple participation options.

## Inclusion checks

- Can students participate in pairs rather than individually if needed?
- Can the teacher read prompts aloud?
- Can reflection be verbal instead of written?
- Can a student opt out of being the primary device user and still participate?
- Are aliases and class summaries handled in a way that avoids embarrassment or unnecessary visibility?

## Accessibility priorities before wider rollout

- Review contrast and readability across the school runtime
- Confirm keyboard accessibility for core classroom controls
- Review wording for cognitive load and age-appropriate clarity
- Test on classroom devices with common browser settings

## Pilot note

Teachers should record any accessibility or inclusion barriers in the implementation fidelity notes so the pilot can be improved before broader adoption.
